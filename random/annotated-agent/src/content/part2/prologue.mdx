## Part 2: The Engineering

Part 1 covered the agent architecture across eight chapters: prompt construction, context management, security, multi-agent coordination, streaming tool execution, memory systems, voice and personality, and future directions. Those are the decisions that make an agent an agent.

This part covers everything else: how two teams solved the same engineering problems in opposite ways. These decisions are not agent-specific, but they shape what the agent feels like to use. The runtime determines startup latency. The concurrency model determines how the UI and the core interact. The terminal framework determines whether streaming text flickers or flows. The platform features determine whether the tool grows into an ecosystem or stays a standalone binary.

Three chapters. The runtime (language, concurrency, state). The terminal UI (framework, streaming display, rich content). The platform layer (transport, persistence, error recovery, and all the features each team built that the other didn't).
