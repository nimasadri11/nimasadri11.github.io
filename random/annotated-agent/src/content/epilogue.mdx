### The Universal Agent CLI

Some things are not bets. They are requirements. Every agent CLI needs: the loop itself, streaming tool execution, a security gate before tool calls run, layered configuration, session persistence, a terminal UI that handles streaming rich content, an extension surface (hooks, MCP, skills), some form of multi-agent coordination, and memory that persists across sessions. Both systems arrived at all of these independently. That convergence is the spec.

The open questions are where reasonable teams will disagree for years. Sandbox or permissions. Few tools or many. Immediate-mode or declarative UI. Tree topology or flexible teams. Implicit memory or active memory management. Server-side or client-side caching. Proactive or reactive compaction. These choices follow from who your user is, whether anyone is watching, and whether the agent runs once or builds a relationship over weeks. There is no universal answer. Pick the defaults that match your most common session, and make the other mode reachable.

### The Minimum Viable Agent Loop

```
loop:
  response = call_model(messages)
  tool_calls = parse_tools(response)
  if no tool_calls: return response
  for call in tool_calls:
    if not permitted(call): skip
    result = execute(call)
    messages.append(result)
```

Everything else in both codebases exists to make this loop safe, fast, and usable.
